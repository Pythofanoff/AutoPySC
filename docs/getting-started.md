## Подробнее о конечной информации: 
Skipped - Счётчик пропущенных файлов если в файле конфигурации параметры группы `REPLACE_EXISTS` == False и в папке проекта есть файлы или папки с таким же названием какие создаёт apysc. <br>
Created - Счётчик созданных файлов, если в папке проекта нету похожих по названию файлов/папок. <br>
Modified - Счётчик изменённых папок/файлов если параметр группы `REPLACE_EXISTS` == True для каждого файла и папки которые имеют одинаковые названия. <br>

## Подробнее о файле конфигурации:
**Файл конфигурации можно открыть командой `apysc open`**, не пугаемся, тут не сложно разобраться и достаточно вчитаться в названия параметров. Нам нужны всего лишь эти строки, все остальные это зависимости для библиотеки, они нам не нужны:

```
NAME_OF_PROJECT = 'MyProject' # DEFAULT = 'MyProject'
AUTHOR = 'DEV' # DEFAULT = 'DEV'
VERSION = 'v0.0.1' # DEFAULT = 0.0.1
DESCRIPTION = f'{NAME_OF_PROJECT} {VERSION} by {AUTHOR}' # DEFAULT = f'{NAME_OF_PROJECT} {VERSION} by {AUTHOR}'
LICENSE = 'MIT' # DEFAULT = 'MIT' 
EMAIL = 'youremail@example.com' # DEFAULT = 'youremail@example.com'
HOMEPAGE = f'https://github.com/{AUTHOR}' # Default = f'https://github.com/{AUTHOR}'
REPOSITORY = f'https://github.com/{AUTHOR}/{NAME_OF_PROJECT}' # Default = f'https://github.com/{AUTHOR}/{NAME_OF_PROJECT}'

LANGUAGE = 'py' # DEFAULT = 'py'
VENV = 'pyvenv' # DEFAULT = 'pyvenv'
ARCHITURE = 'standart' # DEFAULT = 'standart'

REPLACE_EXISTS_FILE = False # DEFAULT = False
REPLACE_EXISTS_FOLDERS = False # DEFAULT = False
QUIET_LAUNCH = False # DEFAULT = False 
```

Пройдёмся по порядку:

**`NAME_OF_PROJECT = 'MyProject' # DEFAULT = 'MyProject'`** - Параметр отвечает за название проекта, название будет участвовать в создании папки по пути src/MyProject и в запись информации о проектк в файл pyproject.toml, то что в кавычках - название проекта, # DEFAULT = 'MyProject' - это комментарий, небольшое напоминание о том какое было название по умолчанию, его не стоит трогать, оно нигде не участвует, так же это будет касаться и других похожих комментариев. <br>

**`AUTHOR = 'DEV' # DEFAULT = 'DEV'`** - Параметр отвечает за имя/никнейм/псевдоним/произвище автора, и будет подставляться в файл LICENCE (правообладатель) и в файл pyproject.toml как создатель. <br>

**`VERSION = 'v0.0.1' # DEFAULT = 0.0.1`** - Параметр отвечает за версию проекта, это версия подставится в файл pyproject.toml, и это начальная версия, позже по ходу работы над вашим проектом, вам нужно будет менять версию уже в pyproject.toml. <br>

**`DESCRIPTION = f'{NAME_OF_PROJECT} {VERSION} by {AUTHOR}' # DEFAULT = f'{NAME_OF_PROJECT} {VERSION} by {AUTHOR}'`** - Параметр отвечает за описание проекта. Описание будет записано в pyproject.toml и в файл README.md

**`LICENSE = 'MIT' # DEFAULT = 'MIT'`** - Параметр отвечает за то, какую лицензию будет использовать ваш проект. Подробнее об лицензиях можно почитать например [Here](https://en.wikipedia.org/wiki/Open-source_license). От выбора лицензии зависит то как и кто будет использовать ваш продукт. В зависимости от выбора лицензии в параметре, в файл LICENCE в корневой директории вашего проекта запишется выбранная лицензия. Ниже приведены поддерживаемые лицензии:  

- MIT = MIT
- Apache License Version 2.0 = ApacheLicenseVersion2

**`EMAIL = 'youremail@example.com' # DEFAULT = 'youremail@example.com'`** - В данный параметр можно передать e-mail который будет подставляться в файл pyproject.toml, он является контактом, чтобы коммюнити вашего продукта предлагала идеи, решения, спонсорство и предложения.

**`HOMEPAGE = f'https://github.com/{AUTHOR}' # Default = f'https://github.com/{AUTHOR}'`** and **`REPOSITORY = f'https://github.com/{AUTHOR}/{NAME_OF_PROJECT}' # Default = f'https://github.com/{AUTHOR}/{NAME_OF_PROJECT}'`** - В HOMEPAGE - ваша ссылка на профиль github, а REPOSITORY - Ссылка на репозиторий проекта (Если есть).

Теперь переходим к главному:

**`LANGUAGE = 'py' # DEFAULT = 'py'`** - Этот параметр пока что нигде не используется, но в будущем он должен создавать структуру проекта под конкретный язык программирования.

**`VENV = 'pyvenv' # DEFAULT = 'pyvenv'`** - Этот параметр отвечает за выбор и создание виртуального окружения. Про виртуальное окружение можно узнать например [Here](https://docs.python.org/3/library/venv.html). На данный момент поддерживается только два значения: `none` and `pyvenv` - первое значение не будет создавать виртуальное окружение, а второй создаст стандартное окружение содержащие в стандартных модулях питона. В будущем планировалось добавить поддержку других виртуальных окружений.

**`ARCHITURE = 'standart' # DEFAULT = 'standart'ARCHITURE = 'standart' # DEFAULT = 'standart'`** - Этот параметр пока что нигде не используется, но в будущем он должен отвечать за создаваемую структуру проекта.

## Теперь настройки которые лучше не трогать:
**`REPLACE_EXISTS_FILE = False # DEFAULT = False`** - Если True, то файлы в папке проекта похожие по названию с теми которые создаёт apysc, будут безвозвратно заменены, если значение будет равно False то apysc не будет заменять файлы, он пропустит их, рекомендую не менять значение на True.

**`REPLACE_EXISTS_FOLDERS = False # DEFAULT = False`** - Если True, то папки в папке проекта похожие по названию с теми которые создаёт apysc, будут безвозвратно заменены, если значение будет равно False то apysc не будет заменять папки, он пропустит их, рекомендую не менять значение на True.

**`QUIET_LAUNCH = False # DEFAULT = False`** - Тихий запуск отвечает за наличие выдаваемой информации в терминале, если включён - то в консоле будет выводиться последний результат о создании структуры проекта, и счётчики, более подробная информация будет сокрыта,рекомендую не менять значение на True, т.к более подробная информации иногда полезнее. 

**Tip: Рекомендуется оставить им значения на False, это гарантирует сохранность ваших файлов и apysc выдаст подробную информацию о процессе создания, эти параметры не достаточно тестировались, возможны сбои, если есть проблемы - немедленно сообщить в Issues Github!** [Click](https://github.com/Pythofanoff/AutoPySC/issues)
